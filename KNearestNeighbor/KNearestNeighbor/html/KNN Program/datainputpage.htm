Data Input Page

Function: The purpose of the page is to allow the selection of the training data and entering the input data along with a k value.

When first entering the Data Input Page, you cannot go on. You can only go back or stay on this page. You must first enter all of the data before you can run the algorithm.

I will be referring to the panels throughout this document. The panels are simply the titles of the different sections on the page. So for instance, on the Data Input Page (step 2), there are 5 panels. There is the Training Information panel at the top, the Input panel in the middle left, the k-NN options in the middle right, the Instructions in the middle bottom, and finally the view training data at the bottom. When I refer to these panels I refer to their contents as well.

Required Fields:

  • Training data must be added. You must click the browse button () and select an acceptable file (*.xlsx). If you have successfully added a file, you should see . If you attempt to select a file that isn't a *.xlsx then it will prompt you to select a valid file type.
  • Once the training data has been entered, the attribute fields will be added dynamically (See Figure 1). All of the attributes need to be entered in each of those boxes. This represents your input data.
  • After entering the input data, you will need to indicate a k value (See Figure 2). You have the option to indicate whether or not the training data and/or the input data is already normalized. If you normalized the data, then we won't normalize it again.
  • Before proceeding, verify your input data and then training data (See Figure 3).

Recommended Steps:

  1. Click browse and add the appropriate data file.
  2. Verify the training data in the Training Information Panel at the bottom.
  3. Enter in your input data.
  4. Enter in an appropriate k value.
  5. If you don't need normalization for your input data, check the "Don't Normalize Input". If you don't need normalization on your training data, check the "Don't Normalize Training Data. Generally these options will be left un-checked.
  6. Click the "Next" button to compute the k-NN algorithm given your data.

Now we will be looking at each of these sections in a bit closer detail.

Training Information Panel

In this panel is a browse button. The button's function is for you to select a file and read in the data. Once a file is correctly read in, the "Number of Attributes:" text will change to reflect how many attributes the program sees. If this number is wrong or doesn't display, then there was something wrong with reading your file. Make sure you have it in the right format.

Input Data Panel

This panel will first appear empty when you first visit this step. That is because the program doesn't know how many attributes you will need to enter until you select a file. Once you select a file, the attribute fields will be dynamically added. If they are not the right amount, then you need to make sure your file is in the correct layout.

All of the fields must contain a continuous value greater than zero to pass validation.

k-NN Options Panel

This panel contains the input box for the k value. The k value must be a natural number.

There is also the option to select "Don't Normalize Training Data" and "Don't Normalize Input". These options are for the case in which the training data selected in the Training Information Panel was already normalized or the input that you gave was already normalized with respect to the entire data set. If either one of those is the case, you should select the appropriate box. There is validation to check the data if it appears normalized or not, but that is not fool proof.

Selecting either one of those boxes when your data isn't normalized will result in wildly inaccurate results and possible run errors.

Instructions Panel

This panel simply gives a brief step-by-step on how to enter information and the order in which it ought to occur.

View Training Data Panel

This panel will first appear empty when you first visit this step. This is because the program doesn't know what the training data is until you have loaded it into the program. Once you select a file, the training data will be populated. If it appears incorrect, then you need to make sure your file is in the correct layout.

Otherwise, this panel provides you the opprotunity to view how the program has interpreted your document. If there is anything incorrect, you should modify your file prior to attempting to compute the algorithm
